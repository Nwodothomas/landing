Payment API Documentation

Introduction

Welcome to the Payment API documentation. This documentation provides details about the endpoints and functionality of our Payment API. Please refer to the sections below for more information.

API Endpoints

Create Payment

Description: Create a new payment transaction.

Request Method: POST

Endpoint: /payments

Request Body: Payment details (e.g., amount, currency, payment method)

Response: Payment transaction ID or confirmation

Request Example:


POST /payments HTTP/1.1
Content-Type: application/json

{
  "amount": 50.00,
  "currency": "USD",
  "payment_method": "credit_card"
}

Response Example:


HTTP/1.1 201 Created
Content-Type: application/json

{
  "transaction_id": "abcd1234",
  "status": "success"
}

Retrieve Payment

Description: Retrieve details of a specific payment transaction.

Request Method: GET

Endpoint: /payments/{payment_id}

Parameters: payment_id - ID of the payment transaction

Response: Payment details (e.g., amount, status, payment method)

Request Example:


GET /payments/abcd1234 HTTP/1.1

Response Example:


HTTP/1.1 200 OK
Content-Type: application/json

{
  "amount": 50.00,
  "status": "success",
  "payment_method": "credit_card"
}

Update Payment

Description: Update the details or status of a specific payment transaction.

Request Method: PUT

Endpoint: /payments/{payment_id}

Parameters: payment_id - ID of the payment transaction

Request Body: Updated payment details (e.g., status, additional information)

Response: Updated payment details or confirmation

Request Example:


PUT /payments/abcd1234 HTTP/1.1
Content-Type: application/json

{
  "status": "pending",
  "additional_info": "Please process the payment as soon as possible"
}

Response Example:


HTTP/1.1 200 OK
Content-Type: application/json

{
  "transaction_id": "abcd1234",
  "status": "pending",
  "additional_info": "Payment update successful"
}

Cancel Payment

Description: Cancel a specific payment transaction.

Request Method: DELETE

Endpoint: /payments/{payment_id}

Parameters: payment_id - ID of the payment transaction

Response: Cancellation confirmation or status update

Request Example:


DELETE /payments/abcd1234 HTTP/1.1

Response Example:


HTTP/1.1 200 OK
Content-Type: application/json

{
  "status": "cancelled",
  "message": "Payment transaction successfully cancelled"
}

List Payments

Description: Retrieve a list of all payment transactions.

Request Method: GET

Endpoint: /payments

Response: List of payment transactions with basic details

Response Example:


HTTP/1.1 200 OK
Content-Type: application/json

{
  "payments": [
    {
      "payment_id": "abcd1234",
      "amount": 100.00,
      "currency": "USD",
      "status": "completed"
    },
    {
      "payment_id": "efgh5678",
      "amount": 50.00,
      "currency": "EUR",
      "status": "pending"
    }
  ]
}

Payment Callback

Description: Receive callbacks or notifications about payment status updates from external payment gateways or processors.

Request Method: POST

Endpoint: /payments/callback

Request Body: Payment callback data

Response: Confirmation of successful processing of the callback

Request Example:


POST /payments/callback HTTP/1.1
Content-Type: application/json

{
  "payment_id": "abcd1234",
  "status": "completed"
}

Response Example:


HTTP/1.1 200 OK
Content-Type: application/json

{
  "message": "Callback processed successfully"
}

Payment Refund

Description: Refund a specific payment transaction.

Request Method: POST

Endpoint: /payments/refund

Parameters:

  • payment_id - ID of the payment transaction

Request Body: Updated payment details (e.g., refund amount, reason)

Response: Refund confirmation or status update

Example Request Body:


{
  "payment_id": "12345",
  "refund_amount": 50.0,
  "reason": "Customer requested a refund"
}

Example Response:


{
  "refund_id": "67890",
  "payment_id": "12345",
  "refund_amount": 50.0,
  "status": "completed"
}

Error Handling

In case of errors, the API will return appropriate HTTP status codes along with error details in the response body. Please refer to the Error Handling section in our API documentation for more information on error codes and their meanings.
